# YMU-A — YMU Attendance

Mobile-first PWA for [Young Musicians Unite](https://youngmusiciansunite.org) teachers to clock in/out at schools with 200 m geofencing, Google Calendar–driven schedules, mandatory post-shift feedback, push notifications, and attendance reports.

Product requirements: [docs/PRD - YMU Attendance APP.md](<docs/PRD - YMU Attendance APP.md>)

## Stack

- **Next.js (App Router, TypeScript, Turbopack)** on Vercel
- **Supabase** — Postgres + RLS, Auth, Edge Functions, pg_cron (all schedulers live here, not Vercel)
- **Serwist** (`@serwist/turbopack`) — service worker, offline support
- **Dexie** — IndexedDB offline queue
- **Leaflet + OpenStreetMap** — clock-in map
- **Web Push (VAPID) + Resend** — notifications and email backup

## Development

```bash
npm install
cp .env.example .env.local   # fill in Supabase keys
npm run dev
```

Production build & serve: `npm run build && npm run start`.

The service worker is compiled from [src/app/sw.ts](src/app/sw.ts) and served at `/serwist/sw.js` by [src/app/serwist/[path]/route.ts](src/app/serwist/%5Bpath%5D/route.ts).

Database migrations live in [supabase/migrations](supabase/migrations); apply with `npx supabase db push` (cloud) or `npx supabase start && npx supabase db reset` (local, requires Docker).

Placeholder app icons are generated by `node scripts/generate-icons.mjs` — replace with real branding before launch.
